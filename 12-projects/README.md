# 12 - Capstone Projects

Now it's time to put everything together! These projects will prove you have the skills of a DevOps Engineer.

## 🚀 Project 1: The Static Website (Beginner)
- **Goal**: Host a simple HTML website.
- **Tech**: AWS S3, CloudFront (CDN), GitHub Actions.
- **Task**: Every time you push a change to GitHub, your website should automatically update.

## 🚀 Project 2: Containerized Web App (Intermediate)
- **Goal**: Deploy a Python/Node.js app with a database.
- **Tech**: Docker, Docker Compose, AWS EC2.
- **Task**: Use Docker Compose to run an app and a database. Automate the build with GitHub Actions.

## 🚀 Project 3: The K8s Cluster (Advanced)
- **Goal**: Deploy a microservices app to Kubernetes.
- **Tech**: Terraform, AWS EKS (or local Minikube), Kubernetes.
- **Task**: Use Terraform to create a K8s cluster. Deploy an app with multiple replicas and a Load Balancer.

## 🚀 Project 4: Full CI/CD Pipeline (Rocketer)
- **Goal**: A "Zero-Downtime" deployment pipeline.
- **Tech**: Everything!
- **Task**: Code -> GitHub -> CI (Tests) -> Docker Build -> Security Scan (Trivy) -> Deploy to K8s -> Monitoring (Grafana).

## 📝 How to Document Your Projects
For each project you build:
1. Create a subfolder (e.g., `12-projects/project-1-static-site`).
2. Add an `architecture.png` (draw a simple diagram).
3. Add a `README.md` explaining:
   - What the project does.
   - The tools used.
   - How to run it.
