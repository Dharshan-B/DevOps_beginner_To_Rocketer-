# 00 - Roadmap and Introduction

DevOps is not a tool, but a **culture** and a set of practices that combine software development (Dev) and IT operations (Ops). The goal is to shorten the systems development life cycle and provide continuous delivery with high software quality.

## 📖 Key Concepts

### 1. The DevOps Lifecycle (The Infinite Loop)
- **Plan**: Defining the business value and requirements.
- **Code**: Development and design of the software.
- **Build**: Building the code and running unit tests.
- **Test**: Automated testing to ensure quality.
- **Release**: Deployment management and versioning.
- **Deploy**: Moving code to production.
- **Operate**: Managing software while it's running.
- **Monitor**: Collecting data on performance and errors.

### 2. Core Principles (CAMS)
- **C**ulture: People and process over tools.
- **A**utomation: Automate everything that can be automated.
- **M**easurement: Data-driven decisions.
- **S**haring: Open communication and transparency.

## 🛠️ The Toolchain
DevOps uses a variety of tools at different stages. This repository will cover:
- **Git** (Source Control)
- **Docker/K8s** (Containerization)
- **Jenkins/GitHub Actions** (CI/CD)
- **Terraform/Ansible** (IaC)
- **Prometheus/Grafana** (Monitoring)
